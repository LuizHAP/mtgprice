<critical_rules>
1. **DO NOT mock `@/db`** in this file. Real DB only (RESEARCH.md anti-patterns, D-10).
2. **DO NOT modify** `src/lib/wishlist/queries.ts` or `src/lib/cards/queries.ts` — production code stays as-is.
3. **FK truncation order is mandatory** in `beforeEach`: `wishlists` first (child), then `cards`, then `users`. Reversing this order will fail with `update or delete on table "cards" violates foreign key constraint` (RESEARCH.md Pitfall 3). The order is: child → parent.
4. **`seedTestUser` MUST be called before `seedTestWishlist`** in every test that uses wishlists (Pitfall 2). The default `userId: 1` in `seedTestWishlist` does NOT auto-create a user — callers must pass the seeded user's id, or pre-seed a user with id=1. Best practice in tests: call `seedTestUser()` first, capture `user.id`, then pass to `seedTestWishlist({ userId: user.id, ... })` and `addCardToWishlist(user.id, ...)`.
5. **FK violation test asserts only that the call rejects, not a specific message.** `addCardToWishlist`'s catch handles code 23505 (unique) but lets 23503 (FK) re-throw as the raw PG error — the message is PG-version-dependent. Use `await expect(...).rejects.toThrow()` with NO message argument (Pitfall 5, D-10).
6. **Biome import order:** internal `@/` imports first (alphabetised), then external (`drizzle-orm`, `vitest`). Single quotes, no semicolons, trailing commas. Run `pnpm biome check --write` after editing.
7. **`test.skip` → `test`:** replace all 6. Keep the describe label `'Wishlist server actions'`. Replace each entire stub body (including comments where they duplicate the new active assertions — feel free to drop the verbose `// Implementation plan:` blocks now that the tests are implemented).
8. **Use explicit `oracleId` per seeded card** when multiple cards are seeded in one test (Pitfall 4) — e.g., `'test-add-card-001'`, `'test-sc-bl-001'`, `'test-sc-bk-001'`, `'test-sc-wk-001'`.
9. **PostgreSQL must be running** for tests to pass. `docker compose up -d postgres` if not.
10. **DO NOT use `Promise.all` for seeding multiple users/cards/wishlists in a single test.** Sequential `await` keeps inserts deterministic and avoids transient unique-constraint collisions on `Date.now()`-derived defaults (Pitfall 4).
