# Gap Context from UAT

**Gap 1: DELETE endpoint returns 500 error**
- Location: `src/lib/wishlist/queries.ts:175`
- Issue: `if (result.length === 0)` fails when `.delete().returning()` returns `null` instead of empty array
- Root cause: Drizzle ORM's `.returning()` can return `null` when no rows match, but code assumes array
- Fix: Add null check before accessing `.length` property

**Gap 2: Duplicate detection doesn't work**
- Location: `src/db/schema/wishlists.ts`
- Issue: Missing unique constraint on (userId, cardId) combination
- Current behavior: Same card can be added multiple times for same user
- Expected behavior: Second insert should fail with PostgreSQL error code 23505 (unique violation)
- Fix: Add `unique("uniqueUserCard").on(table.userId, table.cardId)` to schema, create migration

**Reference from 03-01-SUMMARY.md:**
The `addCardToWishlist` function already handles unique constraint violations (lines 151-158), but the database schema is missing the actual constraint. The function checks for `error.code === '23505'` but this error never occurs because there's no unique constraint.
</context>


    Change to:

    Why: Drizzle ORM's `.delete().returning()` returns `null` when no rows are deleted (not an empty array). Accessing `.length` on `null` throws TypeError: "Cannot read property 'length' of null", causing the 500 Internal Server Error.

    This fix ensures the null case is handled before checking length, preventing the 500 error and allowing proper 404 response.

  <files>src/db/schema/wishlists.ts</files>
  <action>
    Add unique constraint on (userId, cardId) combination to prevent duplicate wishlist entries.

    In src/db/schema/wishlists.ts, after the table definition, add:

    ```typescript
    export const wishlists = pgTable('wishlists', {
      id: serial('id').primaryKey(),
      userId: integer('user_id')
        .notNull()
        .references(() => users.id),
      cardId: varchar('card_id', { length: 255 })
        .notNull()
        .references(() => cards.oracleId),
      addedAt: timestamp('added_at').notNull().defaultNow(),
    }, (table) => ({
      uniqueUserCard: unique().on(table.userId, table.cardId),
    }))
    ```

    Why: This ensures database-level enforcement of the business rule "one user can only add a specific card once". The unique constraint will cause PostgreSQL to throw error code 23505 on duplicate insert attempts, which is already handled by the addCardToWishlist function (lines 151-158 in queries.ts).

    Note: The third parameter to pgTable is a callback that receives the table definition and returns constraint definitions.
  </action>
