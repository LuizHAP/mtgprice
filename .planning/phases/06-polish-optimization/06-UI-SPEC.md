# Phase 6 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Scope Note

Phase 6 is a backend reliability and performance hardening phase. Its three deliverables — retry logic, concurrency control, and Telegram health alerts — produce no new web UI surfaces. The web dashboard already exists and is not redesigned in this phase.

**UI surfaces in scope:**
- Telegram health alert message format (plain text + emoji, delivered via bot)
- Dashboard performance target (< 2s load time — enforced via query optimization, not new UI)
- Optional: `/api/health` endpoint HTTP response body (JSON, no web UI)

**UI surfaces NOT in scope (already implemented in Phase 3):**
- Wishlist page, card grid, price table, search bar, empty state, header, footer

This spec documents the design contract for the Telegram alert surface and confirms the inherited design system for any incidental web touches.

| Token | Value | Usage |

## Color

Inherited from shadcn neutral CSS variables + established semantic usage in Phase 3 components.

| Role | Value | Usage | Source |
|------|-------|-------|--------|
| Dominant (60%) | `hsl(var(--background))` — white (#ffffff in light mode) | Page background, card surfaces | tailwind.config.ts |
| Secondary (30%) | `hsl(var(--card))` / `bg-gray-100` | Card backgrounds, image placeholders, table header hover | CardGrid, PriceTable |
| Accent (10%) | `hsl(var(--primary))` — neutral dark | Primary action buttons only | Button default variant |
| Destructive | `hsl(var(--destructive))` — red | Remove card actions, bulk remove confirmation only | CardGrid `variant="destructive"` |

**Semantic color overrides in use (from existing code — maintain these):**

| Semantic Use | Tailwind Class | Hex Approx | Where |
|--------------|---------------|-----------|-------|
| Best price / price drop (positive) | `text-green-600` | #16a34a | PriceTable best price, CardGrid best price, price trend down |
| Price increase (negative) | `text-red-500` / `text-red-600` | #ef4444 / #dc2626 | CardGrid trend icon up, trend percentage up |
| Stable/neutral trend | `text-gray-400` | #9ca3af | CardGrid trend flat |
| Secondary text | `text-gray-500` / `text-gray-600` | #6b7280 / #4b5563 | Set names, captions, helper text |
| Interactive link | `hover:text-blue-600` | #2563eb | PriceTable card name link hover |

Accent reserved for: Primary action buttons (Button default variant). NOT used for informational displays, prices, or trend indicators — those use semantic green/red/gray directly.

**Telegram alert color encoding:** Telegram does not support color. Semantic meaning is conveyed via emoji prefix:
- `⚠️` — Warning / circuit breaker open
- `✅` — Recovery / circuit breaker closed (if daily summary bonus is added)
- `❌` — Critical failure (if distinguishing severity)

### Telegram Health Alerts (new in Phase 6 — from CONTEXT.md D-03)

| Alert Event | Message Format | Notes |
|-------------|---------------|-------|
| Circuit breaker opens | `⚠️ Circuit breaker aberto: {sourceName} está offline (60s reset). Últimas tentativas falharam.` | Exact format from D-03. Plain text, no parse_mode. Language: Portuguese (consistent with project domain). |
| (Optional) Daily health summary | `✅ Status diário: todas as fontes operacionais. Coletas concluídas: {N}/{total}.` | Discretionary bonus. Same plain text + emoji pattern. |

**Copywriting rules for Telegram alerts:**
- Language: Portuguese (Brazilian) — consistent with project domain language
- Tone: factual and actionable, not alarmist
- Emoji: one per message, prefix position
- Source names: use exact canonical names — `TCGPlayer`, `CardMarket`, `CardKingdom`, `Liga Magic`
- Always include the recovery mechanism ("60s reset") so user knows it is automatic

### Optional `/api/health` JSON Response (discretionary bonus)

If the executor adds this endpoint, the response body must follow:

```json
{
  "status": "ok" | "degraded",
  "sources": {
    "tcgplayer": "ok" | "open",
    "cardmarket": "ok" | "open",
    "cardkingdom": "ok" | "open",
    "ligamagic": "ok" | "open"
  },
  "timestamp": "ISO-8601"
}
```

No web UI renders this — it is for infrastructure monitoring only (e.g., uptime checks).

## Interaction Contracts

### Telegram Health Alert Delivery

| Property | Value |
|----------|-------|
| Trigger | `breaker.on('open', ...)` event in opossum circuit breaker |
| Delivery | `bot.api.sendMessage(Number(chatId), messageText)` — no `parse_mode` |
| Target | `TELEGRAM_CHAT_ID` env var (single-user mode, Phase 1 decision) |
| Failure behavior | Alert failure is caught and logged via Winston. Never propagates. Never crashes circuit breaker pipeline. |
| Frequency | Once per circuit open transition. 60s reset timeout bounds flapping to max 1 alert/minute per source. |
| Language | Portuguese (Brazilian) |

### Dashboard Load Performance Contract

| Metric | Target | Enforcement |
|--------|--------|-------------|
| `GET /api/wishlist` response time | < 2000ms for wishlists up to 200 cards | Query optimization (batch price query or index confirmation). Not a UI concern. |
| Dashboard page First Contentful Paint | < 2000ms | Next.js App Router SSR; existing implementation already SSR. No new UI work needed. |
