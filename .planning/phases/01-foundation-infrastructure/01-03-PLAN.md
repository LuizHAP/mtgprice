<feature>
  <name>Redis-backed token bucket rate limiting</name>
  <files>src/lib/rate-limiter.ts, src/lib/rate-limiter.test.ts</files>
  <behavior>
    Given a rate limit key (e.g., "scryfall"), limit (10), interval (1 second), and tokens to consume (1):
    - If bucket has enough tokens: consume tokens, return { allowed: true, remaining: N }
    - If bucket doesn't have enough tokens: don't consume, return { allowed: false, remaining: 0 }
    - Tokens refill to limit after interval elapses
    - State persists in Redis with TTL = interval
    - Lua script ensures atomic operation (no race conditions)

    Edge cases:
    - First request: bucket starts full (remaining = limit)
    - Exhausted bucket: all requests rejected until refill
    - Concurrent requests: Lua script prevents race conditions
    - Expired keys: bucket refills to limit after interval
  </behavior>
  <implementation>
    1. Create Redis client using ioredis (singleton in src/lib/redis.ts)
    2. Implement checkRateLimit function with Lua script from RESEARCH.md
    3. Add rate limit presets for Scryfall (10 req/sec) and Telegram (100 req/60sec)
    4. Create Next.js middleware that applies rate limiting to /api/external/* routes
    5. Add comprehensive unit tests with Redis mock
  </implementation>
</feature>

Middleware pattern:
```typescript
import { NextRequest, NextResponse } from 'next/server'
import { checkRateLimit, RATE_LIMITS } from '@/lib/rate-limiter'
import { getClient } from '@/lib/redis'

export async function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl

  // Apply rate limiting to /api/external/* routes
  if (pathname.startsWith('/api/external/')) {
    const identifier = request.ip || request.headers.get('x-forwarded-for') || 'anonymous'
    const source = pathname.split('/').pop() || 'default'

    const limit = RATE_LIMITS.SCRYFALL // default, can be dynamic based on source

    const redis = getClient()
    const result = await checkRateLimit(`${source}:${identifier}`, limit.limit, limit.interval, 1)

    if (!result.allowed) {
      return NextResponse.json(
        { error: 'Rate limit exceeded' },
        { status: 429, headers: { 'Retry-After': '60' } }
      )
    }
  }

  return NextResponse.next()
}

export const config = {
  matcher: ['/api/external/:path*']
}
```

This Wires the rate limiter to actual API routes. Phase 2 scraping modules will call /api/external/* endpoints, which will be rate-limited by this middleware.
