<objective>
Implement the bridge between metagame source data (card names) and the local database (oracle_ids): `resolveNamesToOracleIds(names)` POSTs batches of up to 75 names to Scryfall's `/cards/collection` endpoint, parses the resolved card objects, and returns `{ name, oracleId, metadata }` tuples that Plan 04's orchestrator uses to (a) upsert missing cards into the local `cards` table and (b) insert the resolved oracle_ids into `wishlists`. Per RESEARCH.md Pattern 3 + Pitfall 3, the resolver MUST honor the `RATE_LIMITS.SCRYFALL_HEAVY` preset (2 req/s) added by Plan 01 — using the broader `RATE_LIMITS.SCRYFALL` (10 req/s) on `/cards/collection` will trigger HTTP 429.

Purpose: This is the only code path in Phase 5 that calls Scryfall, and it is the single point of correctness for D-06 ("if a metagame card's oracle_id is not yet in the local cards table, fetch its metadata from Scryfall"). By returning the full `ScryfallCard` metadata alongside `oracleId`, Plan 04's orchestrator can call the existing `upsertCards()` from `src/scraper/providers/scryfall.ts` without making a second Scryfall round-trip — cutting orchestrator request count by ~50%.

```typescript
/**
 * Scryfall name → oracle_id batch resolver.
 *
 * Resolves card names harvested from EDHREC/MTGTop8 to Scryfall oracle_ids
 * via the /cards/collection batch endpoint (RESEARCH.md Pattern 3).
 *
 * Rate limit: 2 req/s on heavy endpoints (RESEARCH.md Pitfall 3).
 * MUST use RATE_LIMITS.SCRYFALL_HEAVY (NOT SCRYFALL) — using the regular
 * SCRYFALL preset (10 req/s) will trigger HTTP 429 from Scryfall.
 *
 * Resilience:
 * - Empty input → returns [] without any HTTP call
 * - Single batch failure → logs error, continues with remaining batches
 * - Rate-limit denial → sleeps 500ms and retries once before skipping batch
 * - not_found names → logged as warnings, skipped from result
 *
 * Returns full ScryfallCard metadata alongside oracleId so the orchestrator
 * (Plan 04) can call upsertCards() without a second Scryfall round-trip
 * for D-06 (cards table upsert before wishlist insert).
 */

import axios from 'axios'
import { logger } from '@/lib/logger'
import { RATE_LIMITS, checkRateLimitPreset } from '@/lib/ratelimit/rate-limiter'
import type { ScryfallCard } from '@/scraper/providers/scryfall'

const SCRYFALL_COLLECTION_URL = 'https://api.scryfall.com/cards/collection'
const BATCH_SIZE = 75 // Scryfall documented limit
const RATE_LIMIT_KEY = 'scryfall:collection'
const RETRY_SLEEP_MS = 500

export interface ResolvedCard {
  name: string
  oracleId: string
  metadata: ScryfallCard
}

interface ScryfallCollectionResponse {
  data: ScryfallCard[]
  not_found?: Array<{ name?: string }>
}

function sleep(ms: number): Promise<void> {
  return new Promise((resolve) => setTimeout(resolve, ms))
}

/**
 * Resolve a list of card names to oracle_ids via Scryfall /cards/collection.
 *
 * @param names Card names to resolve (already trimmed and length-capped by Plan 02 fetchers)
 * @returns Array of { name, oracleId, metadata } for each successfully resolved card
 */
export async function resolveNamesToOracleIds(names: string[]): Promise<ResolvedCard[]> {
  // Defensive: filter empty/whitespace names AND short-circuit on empty input
  const validNames = names.map((n) => n.trim()).filter((n) => n.length > 0)
  if (validNames.length === 0) {
    return []
  }

  const results: ResolvedCard[] = []

  for (let i = 0; i < validNames.length; i += BATCH_SIZE) {
    const batch = validNames.slice(i, i + BATCH_SIZE)

    // Rate limit gate (Pitfall 3)
    let rl = await checkRateLimitPreset(RATE_LIMIT_KEY, RATE_LIMITS.SCRYFALL_HEAVY)
    if (!rl.allowed) {
      await sleep(RETRY_SLEEP_MS)
      rl = await checkRateLimitPreset(RATE_LIMIT_KEY, RATE_LIMITS.SCRYFALL_HEAVY)
      if (!rl.allowed) {
        logger.warn(
          `Scryfall rate limit still blocking after retry; skipping batch of ${batch.length} names`,
        )
        continue
      }
    }

    try {
      const { data } = await axios.post<ScryfallCollectionResponse>(SCRYFALL_COLLECTION_URL, {
        identifiers: batch.map((name) => ({ name })),
      })

      for (const card of data.data ?? []) {
        if (card.oracle_id && card.name) {
          results.push({ name: card.name, oracleId: card.oracle_id, metadata: card })
        }
      }

      for (const nf of data.not_found ?? []) {
        logger.warn(`Scryfall could not resolve name: ${nf?.name ?? '(unknown)'}`)
      }
    } catch (error) {
      const message = error instanceof Error ? error.message : String(error)
      logger.error(
        `Scryfall /cards/collection batch failed (${batch.length} names; continuing with next batch): ${message}`,
      )
      // continue — partial success is preferred over abort
    }
  }

  logger.info(`Scryfall resolver: resolved ${results.length}/${validNames.length} names`)
  return results
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Card name strings → Scryfall API | Names sent in POST body; Scryfall sanitizes server-side; we do not embed in URL paths. |
| Scryfall response → process memory | Untrusted JSON parsed and shaped into ResolvedCard[]; size bounded by BATCH_SIZE × per-card payload (~5KB) ≈ 375KB per batch. |
| Rate limit Redis → request gating | If Redis is down, checkRateLimitPreset throws — must not bypass to avoid 429 storm. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-03-01 | Denial of Service | HTTP 429 from Scryfall (rate limit abuse) | mitigate | RATE_LIMITS.SCRYFALL_HEAVY (2 req/s) gates each batch; retry once with 500ms sleep; skip batch on persistent denial. Prevents the 429 storm explicitly called out in RESEARCH.md Pitfall 3. |
| T-5-03-02 | Tampering | Malicious Scryfall response (impossible — public API; OWASP IL2 trust) | accept | Risk tolerance: Scryfall is a known authoritative source for MTG card metadata. Treating responses as authoritative is acceptable. |
| T-5-03-03 | Information Disclosure | axios error messages exposing internal request shape | mitigate | logger.error logs `error.message` only (not full `error.config` or stack). No bearer tokens or DATABASE_URL in error context. |
| T-5-03-04 | Repudiation | No log of which names were sent to Scryfall | accept | logger.info("Scryfall resolver: resolved N/M names") provides counts; per-name audit not required for v1. |
| T-5-03-05 | Tampering | Empty oracle_id from Scryfall response | mitigate | Code defensively filters: `if (card.oracle_id && card.name)` before pushing to results. Empty oracle_ids are silently dropped (would fail FK constraint downstream anyway). |
| T-5-03-06 | DoS | Adversarial input causing many tiny batches (e.g., 1000 single-char names) | mitigate | BATCH_SIZE=75 floor means 1000 names → 14 batches; rate limit gate (2 req/s) → ~7s end-to-end. Bounded resource use. |
| T-5-03-07 | Spoofing | DNS/MITM redirecting api.scryfall.com | accept | Standard HTTPS PKI; consistent with Phase 2 Scryfall calls. No certificate pinning required. |
| T-5-03-08 | Input Validation | Names containing newlines / control chars sent to Scryfall | mitigate | Plan 02 fetchers already trim; this resolver re-applies `.trim()` and filters empty. Names are placed in JSON body (no URL encoding concerns). |
