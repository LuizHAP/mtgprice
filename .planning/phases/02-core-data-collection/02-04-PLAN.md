### From Phase 2 Decisions (02-CONTEXT.md)

**Locked Decisions:**
- Fetch order: Liga Magic first (BR prices critical), then parallel TCGPlayer/CardMarket/CardKingdom
- Circuit breaker: Track failure rate per source, stop if >50% failures
- Error handling: Individual card failures log and skip, don't block entire source
- Smart refresh: Only fetch if >8 hours since last successful fetch per card/source
- Currency conversion: Apply before storing in database (all prices as BRL)
- Schedule: 2-3x daily checks (NOT real-time per NOTIF-03)

**Fetch Orchestration Pattern from RESEARCH.md:**
```typescript
import PQueue from 'p-queue'

async function fetchAllPrices(cardIds: string[]) {
  // Phase 1: Liga Magic (sequential, rate-limited)
  const ligaQueue = new PQueue({ concurrency: 1 })
  for (const cardId of cardIds) {
    await ligaQueue.add(() => fetchLigaMagicPrice(cardId))
  }

  // Phase 2: International sources (parallel)
  const intlQueue = new PQueue({ concurrency: 3 })
  await Promise.all([
    ...cardIds.map(id => intlQueue.add(() => fetchTCGPlayerPrice(id))),
    ...cardIds.map(id => intlQueue.add(() => fetchCardMarketPrice(id))),
    ...cardIds.map(id => intlQueue.add(() => fetchCardKingdomPrice(id)))
  ])
}
```
