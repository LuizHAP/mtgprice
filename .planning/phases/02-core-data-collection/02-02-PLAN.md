# Phase 2 Plan 02: International Price Sources with Circuit Breaker

## Objective

Implement price fetching for the three international sources (TCGPlayer, CardMarket, CardKingdom) using a hybrid API-first/scraping-fallback approach, wrapped in Opossum circuit breakers to prevent cascading failures. This plan completes the price collection layer, enabling multi-source price comparison.

Purpose: Fetch prices from US and European sources with fault tolerance via circuit breakers
Output: Three functional price fetchers returning USD/EUR prices, circuit breaker infrastructure

## Context

### From Phase 2 Decisions (02-CONTEXT.md)

**Locked Decisions:**
- Hybrid approach: API-first, scraping fallback if no API/API limited/API fails
- Circuit breaker pattern: Track failure rate per source, stop source if >50% failures
- Error handling: Individual card failures log and skip, don't block entire source
- Conservative rate limiting: 80% of documented limits (safety buffer)
- TCGPlayer: 50 req/min documented → use 40 req/min
- CardMarket: Unknown → use 40 req/min conservative
- CardKingdom: Unknown → use 40 req/min conservative

**Claude's Discretion:**
- Exact API authentication process (TCGPlayer requires application)
- CardMarket/CardKingdom API availability and endpoints

### From Phase 2 Research (02-RESEARCH.md)

**Opossum Circuit Breaker Pattern:**
```typescript
import CircuitBreaker from 'opossum'

const options = {
  timeout: 10000,                   // 10 second timeout
  errorThresholdPercentage: 50,     // Open circuit when 50% fail
  resetTimeout: 60000,              // Try again after 60 seconds
  rollingCountTimeout: 10000,       // 10-second sliding window
  rollingCountBuckets: 10           // 1-second buckets
}

const breaker = new CircuitBreaker(fetchTCGPlayerPrice, options)

// Fallback when circuit is open
breaker.fallback((cardId) => {
  logger.warn(`TCGPlayer unavailable, skipping ${cardId}`)
  return null
})

// Monitor state changes
breaker.on('open', () => logger.warn('TCGPlayer circuit opened'))
breaker.on('halfOpen', () => logger.info('TCGPlayer circuit half-open'))
```

**Key insight:** Circuit breaker prevents cascading failures - one bad source doesn't block all price collection.

**Open Questions from RESEARCH.md:**
- TCGPlayer API authentication process (requires application for credentials)
- CardMarket API rate limits (assumed 50 req/min for planning)
- CardKingdom API availability (likely scraping-only)
